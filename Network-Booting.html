Network-based booting

iPXE for end-users

  • Diskless booting (iSCSI or RAM)
  • Broad ISO support (operating systems & utilities)
  • Enhanced security (Hypervisor / Split-Booting)
  • Community managed repositories
  • Limited wireless support

Benefits for the enterprise

  • Broader firmware / Driver support (alt: xen, citrix, vmware)
  • Rapid update update cycle (alt: vendor service packs)
  • More secure bare-metal tenancy (alt: KVM)
  • More robust pre-boot remote management (alt: iPMI)
  • Non-proprietary out-of-office management (intel: VT-X)

Definitions:

for servers

Term Definition Service
DHCP Provides IP Address UDP - Port 67 & 68 - Traditionally LAN only
TFTP Provides Network Bootloader Port 69 - Small files only - Broad support
PXE Application provides Kernel Handles installation scripts & additional network protocols
Bootloader OS selection screen Windows Boot Manager, Grub, BootX, etc#

for clients

Term Definition Service
Bios Legacy - "Basic Input/Output System Broad support, uniform vendor features, easier to deploy
UEFI Unified Extensible Firmware Interface Limited support, much greater functionality, lock-in issues
Kernel Bootloader, filesystem mounts, device support, RAM Disks,etc# i.e. Linux, Win-PE, Android, Darwin
